02 The operating system

The OS is the manager standing between your programs and the hardware.

An Operating System (OS) is the most important piece of system software: it manages the computer's hardware and resources, and provides a platform on which application software can run. Without it, every single program would need to know how to talk directly to every ALU, disk and screen — impossible in practice. The OS's core jobs:

🧠
Process management

Decides which program gets the CPU, and for how long, switching rapidly between them so many can appear to run "at once".

🗂️
Memory management

Allocates RAM to each running program and reclaims it when the program closes, keeping programs from overwriting each other.

💾
File management

Organises data into files and folders on storage, and controls how programs read and write them.

🖨️
Device management

Talks to input/output hardware (keyboard, mouse, printer) through small programs called drivers.

🔐
Security & access control

Controls who can use the computer and what each user or program is allowed to do.

🖥️
User interface

Provides the way you interact with the computer — typically a Graphical User Interface (GUI) or a Command Line Interface (CLI).

✋
Link back: "Process management" is the OS deciding whose turn it is at the fetch-decode-execute cycle from Lecture 01 — many programs, one CPU, rapidly time-shared.
TAKEAWAY · The OS manages processes, memory, files, devices and security, and gives you a user interface — so applications never have to touch hardware directly.

04 Application software

The software you actually opened the computer for.

Application software (or "apps") are programs designed to help the user perform specific tasks. Unlike system software, you choose to open them, and the computer works perfectly well without any particular one installed.

07 Why it matters

Every program you'll ever write runs on top of someone else's operating system.

Knowing what the OS handles for you — memory, processes, files, devices — explains why programs can crash safely without destroying the whole machine, why closing an app frees up space, and why the same code can (with adjustments) run on Windows, Linux or macOS. It's the layer every later course — operating systems, networks, systems programming — will zoom into.

Debug smarter

"Out of memory", "permission denied", "process not responding" are all OS-level messages — now you know what they mean.

Choose your platform

Windows for wide software support, Linux for servers and development — an early, practical decision in every CS career.

Exam gold

"Differentiate system and application software" and "compare Windows and Linux" are classic questions — you've now got both.
