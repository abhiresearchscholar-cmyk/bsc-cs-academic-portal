00 Memory vs storage

Fast and forgetful, or slow and permanent — you can't have both.

First, a key distinction. Memory (primary) is fast, works directly with the CPU, and is mostly temporary. Storage (secondary) is slower, larger, cheaper and permanent. In fact every kind of memory sits on a ladder — the memory hierarchy — trading speed against size and cost. As you go up it gets faster, smaller and pricier; as you go down, bigger, cheaper and slower. Click a level to explore it.

The higher a level sits, the faster and costlier it is — and the less of it you get. Registers at the very top are inside the CPU; the cloud at the bottom is effectively unlimited but far away.

TAKEAWAY · Up the hierarchy: faster, smaller, costlier (registers → cache → RAM). Down: bigger, cheaper, slower (SSD → cloud).
01 RAM — working memory

RAM is the computer's desk: fast, roomy — and wiped when the lights go out.

RAM (Random Access Memory) is the main working memory. When you open a program, it's copied from storage into RAM so the CPU can reach it quickly. It's called random access because the CPU can read any location just as fast as any other. Crucially, RAM is volatile — it needs power to hold data. Cut the power and everything in it vanishes. Open some apps, then cut the power and watch.

02 ROM — permanent memory

ROM never forgets — even with the power off.

ROM (Read Only Memory) is non-volatile: it keeps its contents without power. It's written once (at manufacture or by special means) and mainly read afterwards. ROM stores the permanent instructions a computer needs to start up — the BIOS/firmware and the bootstrap program. Variants include PROM, EPROM and EEPROM (which can be erased and rewritten electrically). Cut the power below and compare what survives.

03 Cache memory

A tiny, ultra-fast memory that keeps the CPU from waiting on RAM.

Even RAM is slow compared to the CPU. So a small, very fast memory called cache sits between them, holding the data the CPU is most likely to need next. When the CPU wants something: if it's in cache that's a HIT (fast); if not, a MISS — it must fetch from slow RAM (and copy it into cache for next time). Cache comes in levels L1 (smallest, fastest), L2, and L3. Request some items and watch hits and misses.

04 Secondary & external storage

The permanent stuff: disks, drives, and the cloud.

Everything so far was primary memory. Secondary storage keeps your files permanently — bigger, cheaper and slower than RAM, and not accessed directly by the CPU. Compare the four you'll meet most.

TAKEAWAY · HDD = magnetic, moving parts, cheap/large · SSD = flash, no moving parts, fast · USB = portable flash · Cloud = remote, over the internet.

07 Why it matters

The whole hierarchy exists to hide one problem: memory can't be fast, big and cheap at once.

Every computer you'll ever build juggles this trade-off. Understanding the hierarchy tells you why your programs feel fast or slow, why more RAM helps multitasking, why an SSD boots quicker than a hard disk, and why the cloud is convenient but never instant.

Faster programs

Knowing about cache and locality helps you write code that the hardware can keep fast.

Smart choices

RAM for speed, SSD for daily storage, cloud for backup and sharing — pick the right tool.
